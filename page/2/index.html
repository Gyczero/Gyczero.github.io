从“我的天气”看基于用户心智模型的设计

Posted on 2016-09-01

早上起来看见外面刮风，想到已经立秋了，是不是该换件厚衣服。由于街上人少，不知道外面的人穿的什么，就想看今天的温度来决定换不换衣服。因为视觉方面较突出，我之前一直用“我的天气”这个APP，在看温度时，突然发现不同于iOS原生的天气应用，“我的天气“提供”昨天“的温度。被这么一个细节吸引，回忆自己之前看天气的情景，发现这样一个设计很精妙，通过用户的经历让用户更快速理解信息，更好地去影响用户的决策。

需求背后

分析早上的思考路径，不难发现，是从：
天气变化（诱因） -> 犹豫换厚衣服（决策犹豫） -> 街上人少（未寻找到参考系） -> 看天气 -> 对比昨天的温度(决策参考） -> 决定不换衣服
通常人们想要看温度的背后不仅仅是看温度，背后可能看是否换衣服，是否适合出行，晒被子等，这部分需求探索已经很完整了，在很多APP上都有体现。但在做决策时，我认为用户会分为两类：
1.对温度敏感用户
2.对温度不敏感用户

从蚂蜂窝看产品情感与设计

Posted on 2016-08-29

就像一提到民宿就想到Airbnb，一提到运动就想到Keep一样，蚂蜂窝在“一提到行前决策就想到APP”中占据了不可忽视的地位。作为行前决策和自由行的现象级产品，蚂蜂窝在旅游攻略市场成功切下了自己的一块。以UGC攻略为主的蚂蜂窝，是怎样让用户喜爱依赖的呢，因为UGC是很大的一块，本文先从攻略和情感的角度，尝试分析一下蚂蜂窝对于普通用户的情感共鸣设计。

目标用户

分析蚂蜂窝攻略的设计，首先要确定它的目标用户，这里有两个标签：自由行和行前决策。因为老年人喜欢稳定而且又没有精力，一般都跟团游，所以喜欢自由行的人群集中在18~30岁之间。而这些人群中有对自由行的需求就透露着对个性的需求。对于行前决策有需求的人同时又有着对生活品质的追求，他们喜欢精超过于多。这样大致可以想象出一个18~30岁，大学生或白领中产阶级的形象，他们喜欢玩，喜欢有品质的玩。然后我们看看蚂蜂窝是怎样卖情感和攻略的。
